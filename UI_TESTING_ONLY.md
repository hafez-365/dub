# تشغيل واجهة Dub للاختبار فقط

هذا الملف يوضح كيفية تشغيل واجهة المستخدم بدون أي خدمات خارجية.

## المتطلبات

- Node.js 18+
- pnpm (مدير الحزم)

## خطوات التشغيل

### 1. تثبيت المتعلقات

```bash
pnpm install
```

### 2. تشغيل الخادم

```bash
pnpm dev
```

سيتم تشغيل الخادم على المنفذ **8888**

### 3. فتح التطبيق

افتح متصفحك وانتقل إلى:
```
http://localhost:8888
```

## ملاحظات مهمة

### ✅ ما يعمل:
- عرض واجهة المستخدم والمكونات
- التنقل بين الصفحات
- عرض التخطيطات والنمط

### ❌ ما لا يعمل:
- المصادقة والتسجيل (بدون قاعدة بيانات)
- إنشاء الروابط (يتطلب قاعدة بيانات)
- Analytics والإحصائيات
- الدفع و Stripe
- البريد الإلكتروني
- تحميل/إدارة الملفات
- أي عملية تتطلب API خارجي

## إذا حصلت على أخطاء

### خطأ "DATABASE_URL is required"
```
→ عادي! التطبيق يحاول الاتصال بقاعدة البيانات
→ سيتم عرض الواجهة فقط بدون بيانات
→ تجاهل هذا الخطأ واستمر
```

### خطأ "Tinybird" أو "Upstash"
```
→ هذه الخدمات المدفوعة معطلة
→ لا تتعلق بعرض الواجهة
→ يمكنك استمرار الاختبار بدونها
```

## الخطوة التالية

بعد اختبار الواجهة بنجاح:

1. **إضافة Supabase**
   - أنشئ حساب مجاني على [supabase.com](https://supabase.com)
   - انسخ `DATABASE_URL` إلى ملف `.env.local`
   - شغل: `pnpm prisma:push`

2. **تفعيل المصادقة**
   - أضف `NEXTAUTH_SECRET` عشوائي
   - اختبر تسجيل الدخول

3. **إضافة خدمات أخرى تدريجياً**

## الهيكل

التطبيق عبارة عن Monorepo:
```
.
├── apps/
│   └── web/           ← التطبيق الرئيسي (هنا تركز جهودك)
├── packages/          ← مكتبات مشتركة
│   ├── ui/           ← مكونات UI
│   ├── utils/        ← دوال مساعدة
│   └── prisma/       ← الـ ORM
└── package.json
```
